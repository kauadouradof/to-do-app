# To-Do List com HTML e CSS

Projeto desenvolvido para a atividade **“IA no Desenvolvimento de Software – Engenharia de Prompt aplicada ao Ciclo de Vida do Software”**, proposta pelo **Prof. Msc. Alexander Gobbato**.

## 📌 Objetivo do Projeto

O objetivo deste trabalho foi desenvolver uma **To-Do List funcional utilizando apenas HTML e CSS**, com foco no uso da **Engenharia de Prompt** como apoio ao processo de desenvolvimento.

A proposta não foi apenas gerar código, mas também:
- interpretar requisitos;
- estruturar perguntas de forma inteligente para a IA;
- revisar as respostas recebidas;
- justificar tecnicamente cada decisão tomada.

---

## 🎯 Requisitos do Sistema

O projeto deveria conter:

- **Campo de Entrada**: local para digitar a descrição da tarefa;
- **Seletor de Prioridade**: opções de prioridade (Alta, Média e Baixa);
- **Lista de Exibição**: área onde as tarefas aparecem;
- **Ação de Conclusão**: forma de marcar tarefas como finalizadas;
- **Estilização Condicional**: mudança visual de acordo com a prioridade.

---

## 🧠 Engenharia de Prompt Aplicada

Durante o desenvolvimento, a IA foi utilizada como **assistente** e **par de programação**, seguindo os princípios da atividade:

1. **Programação por Pares com IA**  
   A IA sugeriu abordagens e estruturas, mas a decisão final sobre o código foi do estudante.

2. **IA como Assistente**  
   A ferramenta foi usada para explicar conceitos, sugerir melhorias e revisar a organização do código.

3. **Geração Consciente**  
   Todo o código foi lido, entendido e adaptado antes da versão final.

---

## 🏗️ Etapas do Desenvolvimento

O trabalho foi construído seguindo as etapas propostas no desafio:

### 1. Planejamento
Foi feito o entendimento da lógica do sistema e das limitações do uso exclusivo de HTML e CSS.

### 2. Estrutura (Boilerplate)
Foi criada a base do projeto com HTML semântico e organização clara das seções.

### 3. Implementação
Foi desenvolvida a interface visual do To-Do List com estilização condicional por prioridade e marcação de conclusão usando `checkbox`.

### 4. Revisão
O código foi analisado em relação à acessibilidade, clareza, semântica e organização.

---

## 💻 Tecnologias Utilizadas

- **HTML5**
- **CSS3**

> Não foi utilizado JavaScript, respeitando o enunciado da atividade.

---

## ✅ Funcionalidades Implementadas

- Interface de entrada visual para novas tarefas;
- Seletor de prioridade;
- Lista de tarefas com exemplos;
- Marcação de conclusão por meio de `checkbox`;
- Estilização visual por prioridade:
  - **Alta** → vermelho
  - **Média** → amarelo
  - **Baixa** → verde
- Layout responsivo e organizado;
- Estrutura comentada para facilitar a leitura e defesa do projeto.

---

## ⚠️ Limitação Técnica Importante

Como o projeto foi desenvolvido **apenas com HTML e CSS**, não é possível adicionar tarefas digitadas pelo usuário **de forma dinâmica real**, como aconteceria com JavaScript.

Por isso, a solução foi adaptada para:
- representar corretamente a estrutura de uma To-Do List;
- permitir a conclusão visual de tarefas;
- demonstrar estilização condicional;
- respeitar integralmente a restrição da atividade.

Essa limitação foi considerada como parte da reflexão crítica do trabalho.
